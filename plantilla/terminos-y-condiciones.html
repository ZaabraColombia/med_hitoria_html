  12. Procedimiento y parámetros para la importación de historias clínicas
    13.

    Para importar los datos historia clínica donde se abarcarán los conceptos importantes y el paso a paso para establecer el proceso de migración a la plataforma de MEDHISTORIA, a continuación, se establecerá que información es aceptada por la plataforma.

    12.1.1. Tipos de datos aceptados

    • Numérico: Este dato acepta números enteros y decimales, dentro de la importación los números enteros se escriben normalmente y los decimales con el símbolo (.)

    • Texto: Este dato permite almacenar una cadena de caracteres, donde el máximo de caracteres es de 500Mb, dentro de la importación debe estar encerrado entre comillas dobles (“”).

    • Lista: Este dato permite almacenar valores predefinidos, (valor1, valor2, …) donde cada valor tiene un significado para el profesional, dentro de la importación debe estar separado por comas (,).

    • Booleano: Este campo permite guardar un sí o no (Verdadero o falso), dentro de la importación debe tener lo valores cero o uno (0 ,1).

    • Rango: Este dato se comporta igual que un número, la diferencia es que tiene un rango de valor para seleccionar, dentro de la importación es igual al número.

    12.1.2 Formato de archivos para la importación

    12.1.2.1. Características de archivos

    ▪ Peso: 2mb (por archivo)

    ▪ Formato csv

    ▪ Separación de los datos (;)

    ▪ Si debe tener la cabecera de los datos (nombre de los campos)

    ▪ El nombre debe ser igual al código de la categoría de una Historia clínica.

    12.1.2.2. Arquitectura del archivo

    ▪ Cabecera: id_card (Número de identificativo del paciente); nombre campo 1; nombre campo 2; …

    ▪ Cuerpo: Número de identificativo del paciente; valor 1; valor 2; …

    12.1.3. Procedimiento de importación

    12.1.3.1. Crear los pacientes en la plataforma. Antes de poder migrar la información es necesario tener a los pacientes en el sistema para poder realizar la acción.

    12.1.3.2. Crear los Modelos de Historia Clínica. Un modelo de historia clínica es el conjunto de toda la información sobre una especialidad, este también es personalizable según la necesidad del profesional.

    12.1.3.3. Crear las categorías de Historia Clínica. Una Historia clínica por lo general tiene un conjunto de información muy amplio, la idea es agrupar esta información en subgrupos donde se denominan categorías, Nota: cada categoría es un archivo de importación.

    12.1.3.4. Crear el concepto de la información (variables) de una historia clínica. Una historia clínica tiene toda clase de información la idea es crear el contenedor de esta información respetando los tipos de datos hablados anteriormente, esto se considera una variable de una historia clínica, el objetivo es establecer que información va a recolectar en una historia clínica personalizable.

    12.1.3.5. Descargar archivo de paso a paso de migración. En la vista de importar información podrá descargar un manual donde le indica cómo crear cada archivo y su estructura para poder importarla en nuestro sistema.

    12.1.3.6. Subir archivos de importación. Se debe subir cada archivo donde se debe seleccionar la categoría de la historia clínica y luego subir el archivo, automáticamente el sistema aplicará la configuración e importará todos los datos, en caso de algún error el sistema no subirá la información del archivo.
